# Architectural decision log

Numbered, append-only. When a decision is reversed, strike it and add
a new entry explaining why — don't rewrite history.

Carried forward from v1:

1. **Profiles over duplication** for SOP variants
   (sop-detailed / sop-checklist / sop-walkthrough).
2. **`.glb` as the 3D intermediate.** Blender CLI is the preferred
   converter; lab-PC RobotStudio Premium handles `.rslib` exports the
   student license can't.
3. **Tool library as YAML + shortcode** (`tools.yml` + the `tool` Lua
   shortcode).
4. **Plain-text everything, git-tracked.**

Established by the v2 reorg:

5. **Initiative → Epic → Item is the structural unit.** Reprioritization
   moves whole trees; link integrity survives via epic-prefixed item IDs.
6. **The book is thesis chapters only.** Everything else renders as
   satellite HTML, surfaced via appendix listings.
7. **Listings are generated, not hand-maintained.** Never edit
   `listing.yml` / `_listing.md`.
8. **Updates are directories** (`WU-<date>-week-NN/{index.qmd, media/}`)
   that cross-reference into the `initiatives/` tree.
9. **git-lfs covers media + CAD + Office docs.** GitHub Pages won't
   serve LFS, so deck videos go via Google Drive embeds.
10. **Dispersal = deterministic stager + judgement skill.**
    `disperse.py` stages; the `/disperse` skill routes with the author
    in the loop.
11. **Three-tier scripts.** Tier 1 `scripts/render/` (pipeline), tier 2
    `scripts/tools/<area>/` (cross-cutting), tier 3
    `initiatives/.../scripts/` (item-specific — a convention; none
    exist yet). PEP 723 inline deps for throwaway probes.
12. **QC at publish time, not on-save.**
13. **Conditional `embed-resources` in satellites.** Hub pages keep
    `_files/` (PDF iframes); deep pages embed (Windows MAX_PATH).
14. **Generated `_listing.md` partials for satellite hubs** (Quarto
    `listing:` front matter is a no-op outside a project render).
15. **Conditional subdirs over reflexive ones.** Thin epics get an
    `experiments/` tree when there's work, not before.
16. **Epic README section convention.** Background → Status log →
    (Resolution when done) → generated listing include.
17. **Items own all media; epics own none.**
18. **Generated `_files.md` inventories** close every item page.
19. **Front-matter drift is gated, not policed by hand**
    (`audit_items.py` in the pre-commit hook + publish gate).

Established by the v3 refactor (2026-06):

20. **Satellite renders are content-hash cached.**
    `render_satellites.py` keys each page on source + generated
    partials + asset stats + parent title; outputs restore from
    `.quarto/satellite-cache/`. Corollary: **every generator writes
    only-if-changed** via `_lib.write_*_if_changed` — an idempotent
    rewrite that bumps bytes/mtime is a cache-busting bug. Anything
    copied into the shared render mirror for all satellites must join
    the global cache key. (See docs/render-pipeline.md.)
21. **Ownership is advisory and git-native.** Every initiative / epic /
    item / SOP / WU carries `owner:` front matter, surfaced on listing
    tables, kanban cards, and change-history headers. No CODEOWNERS,
    no commit blocking, no external service (Wiki.js was evaluated and
    rejected: second content store + a server to run). Revisit
    enforcement only if advisory ownership demonstrably fails with
    multiple committers.
22. **Hybrid revision tables.** Auto-generated git change history
    (`build_revisions.py` → `_revisions.md`, committed-history-only
    semantics) on every owned page; SOPs additionally carry a manual
    engineering `revisions:` block with `approved-by` sign-off,
    rendered by the `{{< revisions >}}` shortcode.
23. **Structure is scaffolded, not hand-rolled.** `new_item.py`
    allocates IDs and emits canonical front matter, then self-verifies
    against `audit_items.py`. The audit also covers epics, initiatives,
    and SOP revision blocks. Canonical status set everywhere:
    `open / in-progress / blocked / done` (schedule emphasis lives on
    the Gantt's `crit` tag, not in `status`).
24. **One command surface.** `thesis.ps1` fronts every workflow
    (render / render-one / publish / new / audit / disperse / board /
    doctor); the underlying python scripts remain directly callable on
    non-Windows. The dashboard board (`_board.md`: epic tables, kanban,
    stale items) is generated by `build_board.py` — front matter is
    the input, never the tables.
